What MangoPanel is built to do

The product spec is centered on real shared hosting operations: account provisioning, website management, mail, files, backups, DNS, and safe admin work.

Control plane

SQLite, migrations, and audit logs

The hosting brain stays small and observable: WAL-mode SQLite, scheduled backups, integrity checks, and logged operations.

Versioned state, auditable actions
Hosting model

One stack per customer account

Each customer gets isolated Docker services for the web stack, file manager, databases, mail relay, cron, SFTP, and tools.

Isolation at the account boundary

Product map

Built around the product spec,
not a generic dashboard.

  • Shared hosting first

    The current product is for shared hosting accounts on one server or a small group of servers. VPS/KVM stays a later module.

  • Desired state plus agent jobs

    The API records desired state in SQLite, then a local node agent executes the privileged work and writes back results.

  • Auditable by design

    Account provisioning, SSL issuance, mailboxes, backups, and site changes all leave a trace the operator can inspect later.
